Financial Modelling

Financial Modeling Journey: Building a 3-Statement Model & Forecasting

Overview:

This project marks the beginning of my journey into financial modeling, guided by the insightful YouTube lectures of Parth Verma. I started by learning the foundational conventions used by industry professionals when building financial models, with a strong focus on dynamism and accuracy.

One of the most important early takeaways was the value of making models dynamic rather than hardcoded. This approach improves flexibility and reduces the likelihood of errors, making the model easier to maintain and scale.

Key Features:

  • 3-Statement Financial Model:
    Using financial data sourced from screener.in, I built a fully linked and dynamic 3-statement financial model (Income Statement, Balance Sheet, and Cash Flow Statement). All figures in the model are connected to a central data sheet, following best practices to avoid hardcoding.
  • Forecasting with Linear Regression:
    Implemented linear regression techniques to forecast key financial metrics like revenue and net profit. Unlike mean-based forecasting, regression accounts for historical trends and outliers, offering more accurate and realistic projections.
  • Industry-Standard Conventions:
    Emphasized clean structuring, clear formatting, and proper assumptions management to align with professional modeling standards.

What I Learned:

  • The power of dynamic linking in Excel to enhance model integrity
  • How to source and prepare real financial data for analysis
  • Application of linear regression for forecasting financial trends
  • Best practices for creating scalable and auditable models

Outcome:

This hands-on experience helped me understand how to build robust financial models and make data-driven business decisions. The 3-statement model I developed will serve as a foundation for future analysis, including ratio analysis and valuation modeling.

SQL Data Cleaning Project

Overview:

This project marked the beginning of my journey into SQL and data cleaning. It focused on the foundational techniques needed to prepare datasets for analysis, ensuring that I could work with clean, structured, and accurate data. One of the most important lessons was the importance of not working directly with raw data, and instead duplicating tables to maintain the integrity of the original dataset.

Key Features:

  • Removed duplicates, standardised data, removed nulls, and dropped unnecessary columns
  • Utilised advanced SQL techniques including Window Functions, CTEs, Partition By, and TRIM functions
  • Applied core SQL commands like UPDATE, DELETE, and ALTER TABLE to restructure data
  • Ensured data integrity by duplicating the table and working with the copy

What I Learned:

  • How to use multiple SQL techniques to clean and prepare datasets for analysis
  • The value of maintaining the original dataset and working with duplicates to prevent accidental data loss
  • Critical thinking skills when handling missing or inconsistent data

Outcome:

This hands-on SQL project helped me establish a strong foundation in data cleaning, improved my problem-solving abilities, and laid the groundwork for future data analysis tasks. The skills gained from this project will be instrumental in tackling more advanced data manipulation and analysis challenges.
